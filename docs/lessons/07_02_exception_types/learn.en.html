🔍 Common Python Exception Types

Python has many types of exceptions.

Each exception represents what kind of problem occurred.

Let's learn the most commonly used exception types.

ValueError (Value Error)
int("abc")  # ValueError
int("3.14") # ValueError
# When the value format is incorrect
Raised when the type is correct but the value is inappropriate.
Example: Trying to convert a string to a number, but the string is not a valid number format.
TypeError (Type Error)
"hello" + 5  # TypeError
len(42)      # TypeError
# When the type is wrong
Raised when an operation is performed on the wrong type.
Example: Trying to add a string and a number together.
IndexError (Index Error)
lst = [1, 2, 3]
lst[5]  # IndexError
# When accessing outside list bounds
Raised when accessing an index that doesn't exist in a list or string.
KeyError (Key Error)
d = {"name": "Taro"}
d["age"]  # KeyError
# When accessing a key that doesn't exist in a dict
Raised when accessing a key that doesn't exist in a dictionary.
ZeroDivisionError (Zero Division Error)
10 / 0  # ZeroDivisionError
10 // 0 # ZeroDivisionError
# When dividing by zero
Raised when attempting to divide by zero.

🔀 Using Multiple except Clauses

You can write multiple except clauses to handle different types of errors.

Multiple except clauses
def divide(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        print("Cannot divide by zero")
        return None
    except TypeError:
        print("Only numbers are allowed")
        return None
You can write separate handling for each exception type.
Only the first matching except clause is executed.
Catching multiple exceptions in one except
try:
    result = int("abc") / 0
except (ValueError, ZeroDivisionError):
    print("A ValueError or ZeroDivisionError occurred")
Using except (Error1, Error2): with a tuple lets you handle multiple exceptions in a single except clause.

🌐 Catching All Exceptions with Exception

Exception is the parent class of almost all exceptions.

Writing except Exception: will catch almost any exception.

Using Exception
try:
    x = int("abc")
except Exception as e:
    print(f"Some error: {type(e).__name__}: {e}")

# Output: Some error: ValueError: invalid literal...
type(e).__name__ gets the name of the exception type.
However, specifying concrete exception types makes for better code.
Exception order (specific first)
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Divided by zero")  # this runs
except Exception:
    print("Other error")  # not reached
When there are multiple except clauses, they are checked from top to bottom.
Write specific exceptions first, and generic ones like Exception at the bottom.

💡 Summary

  • ValueError: Value format is incorrect
  • TypeError: Operation performed on the wrong type
  • IndexError: Accessed outside the bounds of a list
  • KeyError: Accessed a key that doesn't exist in a dictionary
  • ZeroDivisionError: Divided by zero
  • Multiple except clauses can handle different exception types
  • Exception catches almost all exceptions
