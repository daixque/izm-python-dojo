🔍 Pythonの主な例外の種類

Pythonにはさまざまな種類の例外があります。

それぞれの例外は、どんな問題が起きたかを表しています。

よく使う例外の種類を覚えておきましょう。

ValueError（値エラー）
int("abc")  # ValueError
int("3.14") # ValueError
# 値の形式が間違っている場合
正しい型だけど値が不正な場合に発生します。
例：文字列を数値に変換しようとしたが、数値の形式ではない。
TypeError（型エラー）
"hello" + 5  # TypeError
len(42)      # TypeError
# 型が間違っている場合
型が間違っている操作をしたときに発生します。
例：文字列と数値を足し算しようとした。
IndexError（インデックスエラー）
lst = [1, 2, 3]
lst[5]  # IndexError
# リストの範囲外を参照した場合
リストや文字列の存在しないインデックスにアクセスしたときに発生します。
KeyError（キーエラー）
d = {"name": "太郎"}
d["age"]  # KeyError
# 辞書に存在しないキーを参照した場合
辞書に存在しないキーにアクセスしたときに発生します。
ZeroDivisionError（ゼロ除算エラー）
10 / 0  # ZeroDivisionError
10 // 0 # ZeroDivisionError
# 0 で割り算した場合
0 で割り算しようとしたときに発生します。

🔀 複数の except を使う

複数の種類のエラーに対応するために、except を複数書くことができます。

複数の except
def divide(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        print("0 で割ることはできません")
        return None
    except TypeError:
        print("数値以外は使えません")
        return None
それぞれの例外に対して別々の処理を書けます。
最初にマッチした except だけが実行されます。
1つの except で複数の例外をまとめる
try:
    result = int("abc") / 0
except (ValueError, ZeroDivisionError):
    print("ValueError か ZeroDivisionError が発生しました")
except (エラー1, エラー2): のように、タプルで複数の例外を1つの except でまとめて扱えます。

🌐 Exception で全ての例外をキャッチする

Exception はほぼ全ての例外の親クラスです。

except Exception: と書くと、どんな例外でもキャッチできます。

Exception を使う
try:
    # 何かの処理
    x = int("abc")
except Exception as e:
    print(f"何かのエラー: {type(e).__name__}: {e}")

# 出力: 何かのエラー: ValueError: invalid literal...
type(e).__name__ でエラーの種類の名前を取得できます。
ただし、具体的なエラーの種類を指定する方が、より良いプログラムになります。
例外の優先順位（具体的なものを先に）
try:
    result = 10 / 0
except ZeroDivisionError:
    print("0 で割りました")  # こちらが実行される
except Exception:
    print("その他のエラー")  # 実行されない
複数の except がある場合、上から順に確認されます。
具体的な例外を上に書き、Exception などの汎用的なものは下に書きましょう。

💡 まとめ

  • ValueError: 値の形式が正しくない
  • TypeError: 型が間違っている操作をした
  • IndexError: リストなどの範囲外にアクセスした
  • KeyError: 辞書に存在しないキーにアクセスした
  • ZeroDivisionError: 0 で割り算した
  • 複数の except でそれぞれの例外に対応できる
  • Exception でほぼ全ての例外をキャッチできる
