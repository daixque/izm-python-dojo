⚙️ Default Arguments

You can set default values for function parameters.

Parameters with default values can be omitted when calling the function. If omitted, the default value is used.

Defining and Using Default Arguments
def greet(name, greeting="Hello"):
    print(f"{greeting}, {name}!")

greet("Alice")               # Hello, Alice!
greet("Bob", "Good morning")  # Good morning, Bob!
Set a default value using = like greeting="Hello".
When you call greet("Alice") without a second argument, "Hello" is used.
When you pass a value like greet("Bob", "Good morning"), it overrides the default.
Multiple Default Arguments
def power(base, exponent=2):
    return base ** exponent

print(power(3))     # 9 (3 squared)
print(power(3, 3))  # 27 (3 cubed)
print(power(2, 10)) # 1024 (2 to the 10th)
exponent=2 is the default value.
Without specifying, it squares; with a value, it raises to any power.

🏷️ Keyword Arguments

When calling a function, you can pass values by specifying the parameter name.

These are called keyword arguments. They are useful because the order doesn't matter.

Calling with Keyword Arguments
def introduce(name, age, city):
    print(f"{name}, age {age}, from {city}")

# Normal call (in order)
introduce("Alice", 13, "New York")

# Keyword arguments (any order)
introduce(age=13, city="New York", name="Alice")
Using keyword arguments allows you to pass arguments in any order.
It also makes the code more readable by showing which value goes to which parameter.
Combining Default and Keyword Arguments
def create_profile(name, age, role="student"):
    return f"{name} (age {age}) - {role}"

print(create_profile("Alice", 13))
# Alice (age 13) - student

print(create_profile("Mr. Smith", 35, role="teacher"))
# Mr. Smith (age 35) - teacher
Combining default arguments and keyword arguments makes flexible functions.

📦 Multiple Return Values

Python functions can return multiple values at once using return.

Multiple values are returned together as a tuple.

Returning Multiple Values
def min_max(numbers):
    return min(numbers), max(numbers)

minimum, maximum = min_max([3, 1, 4, 1, 5, 9])
print(f"Min: {minimum}")  # Min: 1
print(f"Max: {maximum}")  # Max: 9
return min(numbers), max(numbers) returns two values.
You can unpack them with minimum, maximum = ....
Receiving as a Tuple
def divide_with_remainder(a, b):
    quotient = a // b
    remainder = a % b
    return quotient, remainder

result = divide_with_remainder(17, 5)
print(result)     # (3, 2)
print(result[0])  # 3 (quotient)
print(result[1])  # 2 (remainder)
The return value is a tuple (a group of multiple values).
Access each value with result[0] or result[1].

🔭 Variable Scope

Scope refers to the area where a variable can be used.

Variables created inside a function (local variables) can only be used within that function.

Local vs Global Variables
message = "global variable"  # outside function

def show_scope():
    local_var = "local variable"  # inside function
    print(local_var)  # OK
    print(message)    # OK (can read outer variable)

show_scope()
# print(local_var)  # Error! Can't use outside
Local variable: Created inside a function. Only accessible within that function.
Global variable: Created outside a function. Readable from anywhere.
Local variables disappear when the function ends.

📖 Docstrings

You can write a description enclosed in """ right after the function definition. This is called a docstring.

Documenting the purpose and usage of a function makes your code much more readable.

Writing a Docstring
def add(a, b):
    """Return the sum of two numbers.

    Args:
        a: The first number
        b: The second number

    Returns:
        The sum of a and b
    """
    return a + b

print(add.__doc__)  # Display docstring
Write the function description inside """...""".
Access the docstring with function_name.__doc__.

💡 Summary

  • Default arguments def func(x, y=10): create optional parameters
  • Keyword arguments func(y=5, x=3) let you pass arguments in any order
  • return a, b returns multiple values as a tuple
  • Local variables are only valid inside the function; global variables can be read from anywhere
  • Docstrings document the function's purpose and make code more readable
