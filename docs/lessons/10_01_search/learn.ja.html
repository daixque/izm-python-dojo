探索アルゴリズム

📚 説明
English

🔍 探索アルゴリズムとは

探索アルゴリズムとは、リスト（配列）の中から特定の値を見つけるための方法（手順）のことです。

たとえば、電話帳から特定の名前を探したり、棚から特定の本を探したりすることは、日常的な「探索」の例です。

コンピュータでは、大量のデータから素早く目的のデータを見つけるために、効率的な探索アルゴリズムが必要です。

今回は代表的な2つの探索方法を学びます：線形探索と二分探索です。

📋 線形探索（せんけいたんさく）

線形探索とは、リストの先頭から順番に1つずつ調べていく方法です。

探したい値と一致するものが見つかればそのインデックス（位置）を返し、最後まで見つからなければ -1 を返します。

シンプルで確実な方法ですが、リストが長いほど時間がかかります。

線形探索の実装
def linear_search(items, target):
    for i in range(len(items)):
        if items[i] == target:
            return i  # 見つかった！インデックスを返す
    return -1  # 見つからなかった

numbers = [3, 1, 4, 1, 5, 9, 2, 6]
print(linear_search(numbers, 5))  # 4
print(linear_search(numbers, 7))  # -1
range(len(items)) で 0 から最後のインデックスまでループします。
一致したらすぐにインデックスを返し、ループを終了します。

📖 二分探索（にぶんたんさく）

二分探索とは、ソート済み（並び替え済み）のリストを使って、毎回探索範囲を半分に絞り込む方法です。

辞書で言葉を調べるときのことを想像してみてください。「ち」という行を探すとき、辞書の真ん中を開いて「な行」だったら前半を、「あ行」だったら後半を探しますよね。これが二分探索のイメージです。

具体的な手順：

  1. リストの真ん中の要素を見る
  2. 目的の値より大きければ、前半を無視して後半を探す
  3. 目的の値より小さければ、後半を無視して前半を探す
  4. 一致したら見つかり！一致するものがなければ -1
二分探索のイメージ
# ソート済みリストで 7 を探す
items = [1, 3, 5, 7, 9, 11, 13]
# インデックス: 0 1 2 3 4 5 6

# ステップ1: 真ん中 (index=3) = 7 → 一致！
# 答え: index = 3

# もし 11 を探すなら...
# ステップ1: 真ん中 (index=3) = 7 → 11 > 7 なので後半を探す
# ステップ2: 後半の真ん中 (index=5) = 11 → 一致！
# 答え: index = 5
二分探索は毎回探索範囲を半分にします。
100万個のデータでも最大20回程度で見つかります！

⏱️ O記法（オーダー記法）で速さを比べる

O記法（Big-O記法）とは、アルゴリズムの速さをざっくり表す書き方です。

「n個のデータに対して、どれくらいの処理が必要か」を表します。

  • O(n) ：データが n 個あると、最大 n 回処理が必要。データが2倍になると時間も2倍。
  • O(log n)：データが n 個あっても、処理回数はずっと少ない。データが2倍になっても時間はほんの少し増えるだけ。
線形探索 vs 二分探索の速さ比較
# データ数と最大探索回数の比較
#
# データ数 | 線形探索 O(n) | 二分探索 O(log n)
# ---------|--------------|------------------
# 10 | 10 | 4
# 100 | 100 | 7
# 1000 | 1000 | 10
# 1000000 | 1000000 | 20
#
# 1,000,000個のデータから探す場合：
# 線形探索: 最大 1,000,000 回
# 二分探索: 最大 20 回！（ただしソート済みが必要）
二分探索はとても速いですが、必ずソート済みのリストが必要です。
ソートされていないリストには線形探索を使います。

💡 まとめ

  • 線形探索：先頭から順に調べる。ソート不要。計算量 O(n)。
  • 二分探索：ソート済みリストを半分ずつ絞り込む。計算量 O(log n)。
  • O記法：アルゴリズムの速さを表す記法。O(n) より O(log n) の方が速い。
  • データがソートされているなら二分探索が断然速い！
