🔍 What are Search Algorithms?

A search algorithm is a method (a step-by-step procedure) for finding a specific value in a list.

For example, looking up a name in a phone book or finding a book on a shelf are everyday examples of "searching."

In computer programs, we need efficient search algorithms to quickly find the right data from large amounts of information.

In this lesson we'll learn two classic search methods: linear search and binary search.

📋 Linear Search

Linear search checks each element in a list one by one from the beginning.

If the target value is found, it returns the index (position). If it reaches the end without finding it, it returns -1.

It's simple and reliable, but takes longer as the list gets bigger.

Implementing Linear Search
def linear_search(items, target):
    for i in range(len(items)):
        if items[i] == target:
            return i  # found! return index
    return -1  # not found

numbers = [3, 1, 4, 1, 5, 9, 2, 6]
print(linear_search(numbers, 5))  # 4
print(linear_search(numbers, 7))  # -1
range(len(items)) loops from index 0 to the last index.
When a match is found, we immediately return that index and stop the loop.

📖 Binary Search

Binary search works on a sorted list and narrows down the search range by half each time.

Think of looking up a word in a dictionary. If you want "rabbit", you open to the middle. If you land on "monkey", you know to look in the second half. That's exactly how binary search works!

The steps:

  1. Look at the middle element of the list
  2. If the target is larger, ignore the first half and search the second half
  3. If the target is smaller, ignore the second half and search the first half
  4. If it matches — found! If no match remains — return -1
Binary Search Walkthrough
# Searching for 7 in a sorted list
items = [1, 3, 5, 7, 9, 11, 13]
# index: 0 1 2 3 4 5 6

# Step 1: middle (index=3) = 7 → match!
# Answer: index = 3

# Searching for 11...
# Step 1: middle (index=3) = 7 → 11 > 7, search right half
# Step 2: middle of right half (index=5) = 11 → match!
# Answer: index = 5
Binary search halves the search range at every step.
Even with 1,000,000 items it finds the answer in at most 20 steps!

⏱️ Comparing Speed with Big-O Notation

Big-O notation is a way to roughly describe how fast an algorithm is.

It expresses "how much work is needed for n items of data."

  • O(n): For n items, up to n operations needed. Double the data → double the time.
  • O(log n): Even for n items, the number of operations stays small. Doubling the data only adds one extra step!
Linear Search vs Binary Search Speed
# Comparison of data size vs max steps
#
# Data size | Linear O(n) | Binary O(log n)
# ----------|-------------|----------------
# 10 | 10 | 4
# 100 | 100 | 7
# 1000 | 1000 | 10
# 1000000 | 1000000 | 20
#
# Searching 1,000,000 items:
# Linear search: up to 1,000,000 steps
# Binary search: up to 20 steps! (but list must be sorted)
Binary search is incredibly fast, but requires a sorted list.
For unsorted lists, use linear search.

💡 Summary

  • Linear search: checks each item one by one. No sorting needed. O(n).
  • Binary search: halves the search range each time. Requires sorted list. O(log n).
  • Big-O notation: describes algorithm speed. O(log n) is much faster than O(n).
  • If your data is sorted, binary search wins by a huge margin!
