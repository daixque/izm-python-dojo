🗄️ What are Databases and Tables?

A database is like a warehouse where you can store and organize large amounts of data. Almost every smartphone app and web service uses a database to manage user information and content.

Inside a database, data is stored in tables. A table works like a spreadsheet — it has rows (records) and columns (fields).

🔤 What is SQL?

SQL (Structured Query Language) is the language used to give instructions to a database — adding, retrieving, updating, and deleting data.

Importing sqlite3 and connecting
import sqlite3

# Create an in-memory database (gone when program ends)
conn = sqlite3.connect(":memory:")
cursor = conn.cursor()
sqlite3 is part of Python's standard library — no installation needed. Using ":memory:" creates a database that lives only in RAM, which is perfect for learning and testing.
Creating a table (CREATE TABLE)
cursor.execute(
"CREATE TABLE users (id INTEGER, name TEXT, age INTEGER)"
)
CREATE TABLE table_name (column_name type, ...) creates a new table. INTEGER means a whole number and TEXT means a string.
Inserting data (INSERT INTO)
cursor.execute(
"INSERT INTO users VALUES (?, ?, ?)",
(1, "Alice", 25)
)
conn.commit()
INSERT INTO table_name VALUES (...) adds one row. The ? marks are placeholders — pass the actual values as a tuple in the second argument. Call conn.commit() to save the changes.
Fetching data (SELECT)
cursor.execute("SELECT * FROM users")
rows = cursor.fetchall()
print(rows)
# [(1, 'Alice', 25), (2, 'Bob', 30)]
SELECT * FROM table_name retrieves all columns and all rows. fetchall() returns the results as a list of tuples.

📝 Basic sqlite3 Workflow

  1. Connect with sqlite3.connect(":memory:")
  2. Get a cursor with conn.cursor()
  3. Run SQL commands with cursor.execute(sql)
  4. Save changes with conn.commit()
  5. Read results with cursor.fetchall()
