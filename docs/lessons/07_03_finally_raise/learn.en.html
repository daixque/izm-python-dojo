🔒 The finally Block

Code in the finally block is always executed, whether or not an error occurs.

It's used for "cleanup" tasks such as closing files or disconnecting from a network.

Basic finally
try:
    print("processing...")
    x = int("abc")  # error occurs
except ValueError:
    print("caught error")
finally:
    print("always runs")

# Output:
# processing...
# caught error
# always runs

✅ The else Block

Code in the else block runs only when no error occurred in the try block.

Using else
try:
    x = int("42")
except ValueError:
    print("conversion error")
else:
    print(f"success: {x}")  # runs only if no error
finally:
    print("done")

# Output:
# success: 42
# done
else: contains code for when no error occurred.
The order is: try → no error → else → finally.

🚀 Raising Exceptions with raise

You can use raise to deliberately raise an error.

This is useful when validating input or signaling that something is wrong.

Basic raise
def check_age(age):
    if age < 0:
        raise ValueError("Age must be 0 or greater")
    return age

check_age(25)   # 25
check_age(-1)   # ValueError: Age must be 0 or greater
raise ValueError("message") raises an exception.
The caller can catch it with try/except.
Catching a raised exception
def check_age(age):
    if age < 0:
        raise ValueError("Age must be 0 or greater")
    return age

try:
    result = check_age(-1)
except ValueError as e:
    print(f"Input error: {e}")

# Output: Input error: Age must be 0 or greater
Exceptions raised with raise can be caught with try/except just like any other exception.

🏗️ Custom Exception Classes

You can create your own exception classes.

Define a class that inherits from Exception.

Defining and using a custom exception
class NegativeAgeError(Exception):
    pass

def check_age(age):
    if age < 0:
        raise NegativeAgeError(f"Age {age} is invalid")
    return age

try:
    check_age(-5)
except NegativeAgeError as e:
    print(f"Custom error: {e}")
class MyError(Exception): pass creates a custom exception.
You can define meaningful error names for your program.

💡 Summary

  • finally: always runs regardless of whether an error occurred
  • else: runs only when no error occurred in the try block
