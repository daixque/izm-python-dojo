finally と raise

📚 説明
English

🔒 finally ブロック

finally ブロックに書いた処理は、エラーが起きても起きなくても必ず実行されます。

ファイルを閉じる、接続を切るなど「後片付け」に使います。

finally の基本
try:
    print("処理中...")
    x = int("abc")  # エラー発生
except ValueError:
    print("エラーをキャッチ")
finally:
    print("必ず実行されます")

# 出力:
# 処理中...
# エラーをキャッチ
# 必ず実行されます
finally: の中の処理は、エラーが起きても起きなくても実行されます。
「どんな場合でも最後にやること」を書きます。
エラーがないときの finally
try:
    print("処理中...")
    x = int("42")  # 成功
except ValueError:
    print("エラーをキャッチ")  # 実行されない
finally:
    print("必ず実行されます")  # 実行される

# 出力:
# 処理中...
# 必ず実行されます
エラーがなくても finally は実行されます。

✅ else ブロック

else ブロックに書いた処理は、try ブロックでエラーが起きなかったときだけ実行されます。

else の使い方
try:
    x = int("42")
except ValueError:
    print("変換エラー")
else:
    print(f"変換成功: {x}")  # エラーがないときだけ実行
finally:
    print("処理完了")

# 出力:
# 変換成功: 42
# 処理完了
else: はエラーが起きなかったときの処理です。
try → エラーなし → else → finally の順に実行されます。

🚀 raise で例外を発生させる

raise を使うと、自分でエラーを発生させることができます。

入力値の検証など、意図的にエラーを出したい場合に使います。

raise の基本
def check_age(age):
    if age < 0:
        raise ValueError("年齢は 0 以上でなければなりません")
    return age

check_age(25)   # 25
check_age(-1)   # ValueError: 年齢は 0 以上でなければなりません
raise ValueError("メッセージ") で例外を発生させます。
呼び出し元では try/except でキャッチできます。
raise した例外をキャッチする
def check_age(age):
    if age < 0:
        raise ValueError("年齢は 0 以上です")
    return age

try:
    result = check_age(-1)
except ValueError as e:
    print(f"入力エラー: {e}")

# 出力: 入力エラー: 年齢は 0 以上です
raise で発生させた例外は、普通の例外と同じように try/except でキャッチできます。

🏗️ カスタム例外クラス

自分だけの例外クラスを作ることもできます。

Exception を継承してクラスを定義します。

カスタム例外の定義と使い方
class NegativeAgeError(Exception):
    pass

def check_age(age):
    if age < 0:
        raise NegativeAgeError(f"年齢 {age} は無効です")
    return age

try:
    check_age(-5)
except NegativeAgeError as e:
    print(f"カスタムエラー: {e}")
class MyError(Exception): pass でカスタム例外が作れます。
自分のプログラムで意味のある名前のエラーを定義できます。

💡 まとめ

  • finally: はエラーの有無にかかわらず必ず実行される
  • else: はエラーが起きなかったときだけ実行される
  • raise 例外クラス("メッセージ") で意図的に例外を発生させられる
  • Exception を継承してカスタム例外を作ることができる
