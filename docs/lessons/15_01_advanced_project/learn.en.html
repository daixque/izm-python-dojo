📋 Project Overview: Grade Management System

You're going to build a system that manages student names and test scores.

  • Register multiple scores for each student
  • Calculate the average score for a specific student
  • Show a ranking of the top N students by average score
  • Raise a meaningful error when an unknown student is searched

This is the kind of tool a real teacher might actually use. Approach it with that mindset!

② sqlite3 (Database)
import sqlite3

conn = sqlite3.connect(":memory:")
cursor = conn.cursor()
cursor.execute(
    "CREATE TABLE students (name TEXT, score REAL)"
)
conn.commit()
sqlite3 is a database library that comes built into Python — no installation needed.
Using ":memory:" stores data in RAM (it disappears when the program ends).
You use SQL statements to add and query data.

💡 SQL Statements Review

This project uses three types of SQL statements.

Inserting Data (INSERT)
cursor.execute(
    "INSERT INTO students VALUES (?, ?)",
    (name, score)
)
? is a placeholder. Values from the tuple in the second argument are substituted in order.
Always use placeholders instead of embedding strings directly — this prevents SQL injection attacks.
Calculating Averages (SELECT AVG)
cursor.execute(
    "SELECT AVG(score) FROM students WHERE name = ?",
    (name,)
)
result = cursor.fetchone()[0]
AVG(score) calculates the average of all matching scores.
WHERE name = ? filters records to a specific student.
fetchone() retrieves one row, and [0] extracts the first column value.
Building a Ranking (GROUP BY / ORDER BY)
cursor.execute(
    "SELECT name, AVG(score) as avg FROM students "
    "GROUP BY name ORDER BY avg DESC LIMIT ?",
    (n,)
)
return cursor.fetchall()
GROUP BY name groups rows by student name.
ORDER BY avg DESC sorts by average score from highest to lowest.
LIMIT ? returns only the top N results.

🗺️ Design First, Then Code

When building something large, resist the urge to start typing immediately. Think about the design first.

For this system, ask yourself:

  • What do I need to store? → Student names and scores
  • What operations are needed? → Add, get average, get ranking
  • Where can errors happen? → When searching for a student who doesn't exist

Having clear answers before writing a single line of code will keep you on track.

🧩 Break Big Problems Into Small Pieces

"Build a grade management system" might sound overwhelming. But broken into steps, it's very manageable.

  1. Start with __init__ — just create the table
  2. Then implement add_student — insert one record at a time
  3. Next, make get_average calculate and return a value
  4. Finally, build top_students to produce the ranking

Test each method as you go. You don't need to get everything right at once.
